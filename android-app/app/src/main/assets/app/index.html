Multi-signal matching

Perceptual frame hashing, color histograms and chroma audio fingerprints are fused for robust detection.

WebWorker-powered

Frame extraction, FFT and cross-similarity run off the main thread so the UI never freezes.

Private by design

Your media never leaves the browser. Everything — decode, fingerprint, compare — is local.

Choosing the right mode

Each mode has different performance and accuracy tradeoffs. Here's our recommendation based on deep analysis.

Audio

Recommended
Performance

Fast

Accuracy

High for music

Best for real-time playback detection. Chroma features are robust to key/tempo changes and audio extraction is ~10× faster than video.

Best for

Intros/outros with reused music, theme songs, voice-overs

Video

Performance

Slow

Accuracy

High for visuals

Frame extraction requires seeking which is DOM-bound and slow. Best when audio was replaced but visuals are identical.

Best for

Silent intros, visual-only logos, audio-swapped content

Combined

Most accurate
Performance

Moderate

Accuracy

Highest

Fuses both signals. Catches matches that either mode alone would miss. Use for offline analysis where accuracy matters more than speed.

Best for

Final verification, ambiguous cases, production analysis

For your playback app: use Audio mode

Audio fingerprinting is fast enough for real-time detection during playback (~10ms per 1-second window). Export an audio signature from Compare mode, then use Detect mode or integrate the signature into your player to auto-skip intros/outros live.

All processing happens locally in your browser.
